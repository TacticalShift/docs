Как добавить картинки?

    • Варианты добавления картинок
      • Ссылка на картинку из репозитория
      • Прямая веб-ссылка
      • Base64

    Общий Mardown синтаксис для картинок:

    ![Image_name](image_URL_or_path "Image tytle")

    Где Image_name - обязательное название картинки, image_URL_or_path - собственно ссылка или путь до картинки, "Image tytle" - опциональное поле для отображение текста при наведении на картинку.

    Варианты добавления картинок

    Ссылка на картинку из репозитория

    Картинки в репозитории хранятся по пути src/media, для добавлении картинки её нужно скопировать в эту папку, а при использовании в документах ссылку формировать как /docs/src/media/название_картинки.расширение. К сожалению, локально проверить работоспособность картинки не получится, только после деплоя. Пример:

    ![Carl](/docs/src/media/Carl.png "Carl")
или для любителей html
<img src="/docs/src/media/Carl.png" alt="Carl" />

    Прямая веб-ссылка

    Способ самый простой, но всегда есть вероятность, что ссылка перестанет быть доступной - картинку перенесут, удалят, etc. Пример:

    ![Шеврон](https://lh7-us.googleusercontent.com/VvbRQd0kaMwY9fVojzfO_fV56qWV82WflilazsuF-ArArLmqSRB2HIRx0kpW34dQkjGG5UOjecQYq6okOVVptNmWoCvAyb2zNOkVbXjX4Bt9oQQmuLdDLbdBsZNYOvolHC7q-c5xrYzGfRXMPvy0wHA)

    Base64

    Способ подразумевает текстовую репрезентацию картинки внутри markdown-документа.

    Первый вариант - embedded html code, а поэтому не рекомендуется для новых статей, так как сильно затрудняет редактирование (примеры в "старых" статьях - documentation_sop_graphics.md).

    Второй вариант - собственно markdown image, который позволяет разделить место вставки картинки и её "определение". Пример (взят из documentation_sop_communications.md):

    ![kompas]
